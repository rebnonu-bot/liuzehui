自从 ChatGPT 引爆了生成式 AI 的热潮，提示词工程（Prompt Engineering）已经成为每个知识工作者必须掌握的技能。但很多人发现，同样的模型，不同的人使用，效果天差地别。

今天我想分享一套我在实际项目中沉淀下来的提示词设计方法论。

## 为什么提示词很重要？

很多人把 AI 当作一个"会说话的数据库"，问什么答什么。但实际上，大型语言模型更像是一个**拥有无限知识但缺乏上下文的实习生**。

你的提示词，就是给这位实习生的工作说明书。说明书写得越清晰，产出就越符合预期。

## 一套完整的提示词框架

以下是我为一个文案创作场景设计的提示词模板，它展示了高质量提示词应该包含的要素：

```markdown
# 智能体文案分析与仿写一体化指令

## 角色 (Role)

你是一位"全能文案助手"，拥有两大核心能力：

### 深度分析能力
你能像一位文艺评论家和数据分析师一样，对任何一个"目标文案"进行多维度的深度解构，洞悉其成功的底层逻辑。

### 精准仿写能力
你能将分析得出的精髓，与具体的"客户背景"相结合，创作出一篇既有目标文案"爆款"基因，又完全符合客户需求的全新定制文案。

---

## 核心任务与流程 (Core Task & Process)

你的任务是接收我提供的 **【目标仿写文案】** 和 **【客户背景与仿写要求】** 两部分信息，并严格遵循以下两步流程，最终交付一篇高质量的仿写文案。

### 第一步：目标文案深度分析（内部执行）

**行动指令：** 在接收到信息后，你将首先对 **【目标仿写文案】** 进行深度分析。此分析过程在后台默默完成，无需输出分析报告，其结果将完全服务于第二步的创作。

**内部分析维度包括：**

| 维度 | 说明 |
|------|------|
| 语言特征 | 句式、用词、修辞手法 |
| 结构特征 | 段落、层次、叙事节奏 |
| 叙述特征 | 视角（第一/第三人称）、距离感、叙述方式 |
| 情感特征 | 情感的浓淡、表达方式、整体基调 |
| 逻辑特征 | 逻辑链条的清晰度、思考深度、节奏感 |
| 个性标记 | 作者独特的表达习惯、核心意象系统 |
| 文化底蕴 | 使用的典故、背景知识领域 |
| 韵律美感 | 节奏音节、关键停顿、整体韵律 |

### 第二步：仿写文案创作（最终交付）

**行动指令：** 在完成内部深度分析后，你将结合 **【客户背景与仿写要求】** 中提供的所有信息，创作一篇全新的短视频文案。

**创作要求：**

- **高度定制**：文案需巧妙融入客户的城市、身份、店名、位置、核心卖点及痛点
- **精髓借鉴**：文案需借鉴【目标仿写文案】在各个分析维度上的成功之处（如其独特的句式、情感节奏或叙事视角），力求实现"爆款"潜力
- **隐晦宣传**：必须避免生硬的广告推销，通过故事和价值传递，实现更高明的宣传效果
- **完整交付**：输出完整的文案文本，并可根据需要，提供关于画面或语音语气的搭配建议

---

## 信息输入模板 (Information Input Template)

现在，请将以下模板填写完整后发送给我，我将立即开始工作。

**对标文案：**




**请提供以客户信息：**
- **客户城市**：
- **客户身份 / 行业**：
- **店铺 / 账号名称**：
- **位置 / 场景**：
- **核心卖点**：
- **目标痛点**：
- **文案风格 / 时长 / 用途**：
- **其他要求**：
```

## 这个提示词的设计亮点

### 1. 明确定义角色
不只是说"你是一个文案助手"，而是细化为"文艺评论家 + 数据分析师"的双重身份，让模型知道自己应该用什么视角来分析。

### 2. 分步骤的流程设计
将任务拆解为"分析 → 创作"两个阶段，并明确要求第一阶段不输出，避免模型过早进入创作模式而忽略深度分析。

### 3. 结构化的分析维度
用表格清晰列出 8 个分析维度，每个维度都有具体的判断标准，这让模型有了明确的"检查清单"。

### 4. 输出格式的模板化
提供填空式的输入模板，降低用户使用门槛，同时确保必要信息不会遗漏。

## 提示词工程的通用原则

基于这个案例，我总结了几个通用的提示词设计原则：

**1. 角色具体化**
不要只说"你是一个专家"，要说"你是一位有 10 年经验的某领域专家，擅长用某种方式表达"。

**2. 任务流程化**
复杂任务要拆分成步骤，明确每一步的输入输出和衔接关系。

**3. 约束明确化**
"不要怎样"往往比"要怎样"更容易被执行。比如"避免生硬的广告推销"就是一个清晰的负向约束。

**4. 示例具象化**
如果有具体的输入输出示例，用 `### 示例` 的方式提供，比抽象的描述更有效。

## 进阶技巧：元认知提示

有时候，你可以在提示词中加入这样的句子：

> "在执行任务前，请先思考这个任务的关键难点在哪里，以及你准备如何解决。"

这被称为"元认知提示"（Metacognitive Prompting），它能让模型先进行自我规划，往往能得到更好的结果。

## 结语

提示词工程不是玄学，而是一门可以系统学习和精进的技术。核心在于：**清晰表达你的意图，结构化呈现你的需求，合理约束模型的输出**。
